# Product Spec

## 一句话说明

File Organizer 是一个安全优先的文件夹整理工具：先生成可审核的整理计划，再按用户确认执行，并保留可回滚记录。

## 目标用户

- 经常整理 Downloads、Desktop、截图、文档归档的个人用户。
- 需要把资料按项目、来源或时间归档的知识工作者。
- 希望批量整理但担心误删、覆盖、误移动的谨慎用户。

## 核心问题

普通整理脚本通常有三个风险：

- 规则不可见，用户不知道文件为什么会被移动。
- 执行不可撤销，出错后很难恢复。
- 冲突处理粗糙，容易覆盖文件或制造混乱。

File Organizer 的产品策略是：先让计划可信，再让执行省心。

## MVP 范围

- 扫描单个文件夹。
- 支持递归扫描。
- 按扩展名分类到目标目录。
- 生成 dry-run 计划。
- 文件名冲突时生成唯一目标名。
- 输出文本和 JSON。
- 跳过隐藏文件，可显式包含。

## V1 范围

- 规则配置文件。
- 操作 journal。
- 执行移动文件。
- 按 journal 撤销。
- 按日期、大小、名称模式和来源路径分类。
- 重复文件检测。

## V2 方向

- 内容摘要和语义分类建议。
- 图形界面。
- 规则调试器。
- 计划差异视图。
- 定时整理和后台监控。

## 非目标

- 不做危险的一键删除。
- 不默认上传用户文件。
- 不在没有 journal 的情况下批量移动。
- 不默认接管系统文件夹。

## 成功标准

- 用户能在执行前清楚看到每个文件的目标位置。
- 同名文件不会被覆盖。
- 所有执行动作可以从 journal 追溯。
- 常见整理任务可以通过规则复用。
- 错误信息能帮助用户修复问题，而不是只显示堆栈。
